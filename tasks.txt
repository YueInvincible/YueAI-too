YueAI current checklist
=======================

Completed and verified
----------------------
- Core now has an initial durable agent-run surface: `SQLiteAgentRunStore`,
  `YueCore.start_agent_run(...)`, JSONL `agents.runs.start/get/list`, and
  `agent.run.*` events.
- Agent runs now also persist structured checklist items and verification
  status/summary through JSONL update methods.
- Agent runs now persist persona/provider snapshots and automatically record
  tool/approval references from runtime events.
- Permission core now has session capability/resource grants exposed through
  JSONL `permissions.capability_grants.get/set` and desktop protocol/mock/runtime
  wrappers; grants cannot bypass the active profile or be changed by `model`.
- Scoped grants now support JSONL revoke plus first lifetime enforcement:
  `once`, `run`, `conversation`, and `session`.
- Permission decisions and denied tool results now carry structured
  `denial_category` plus `resource_scope`; tool activity can distinguish
  missing-scope denials without parsing error text.
- Single active-provider flow is wired through backend and desktop UI.
- Local/OpenAI-compatible and Anthropic provider settings can be edited from the desktop shell.
- Run inspector, tool activity reconnect snapshot, and grouped run summaries are in place.
- Desktop chat surface has been redesigned into a chat-first shell with hidden Ops/Config drawers.
- Ops drawer now contains provider health, tool contract, run inspector, shell grant, and parallel inspect.
- Chat area is now fixed-frame; the temporary Focus chat expansion mode has been removed.
- Active provider drawer now has a dedicated runtime refresh action and readiness strip for reachability/model detection.
- Desktop protocol/state test coverage now locks the active-provider flow, including persisted apply-state through the mock transport.
- Desktop Tauri diagnostic harness now retries packaged bootstrap probes and writes richer stages (`probe_dispatching`, `probe_dispatched`, `probe_timeout`) for NSIS debugging.
- Packaged NSIS desktop path has been re-verified green with the richer diagnostic harness; the log reaches `runtime_bootstrap` and `bridge: spawned | core: started`.
- Packaged NSIS cleanup proof is also green: the app spawns `python -m yue_core ... serve`, and the child exits when the desktop process is terminated.
- JS protocol tests pass.
- Full Python test suite passes.

Known next targets
------------------
- New source-of-truth checklist: `docs/notes/omni_agent_checklist.md`.
- Re-orient product priorities to:
  1. core agent runtime;
  2. desktop UI;
  3. permission/tools;
  4. memory;
  5. plugin ecosystem;
  6. 3D avatar;
  7. release hardening.
- Treat Codex as the reference for agent loop, sandbox/approval, bounded
  context, config layering, tool contracts, and plugin discipline; Yue should
  extend this into desktop-wide assistance.
- Do not write many built-in tools yet. Permission center list/revoke UI is now
  wired. Next add permission center audit preview/recent risky actions and
  stronger denied metadata display, then plugin install/trust lifecycle,
  persona schema, and memory schema.
- Reduce drift between `desktop/src/app.js` and `desktop/src/runtime.js`; note that `desktop/index.html` currently loads `runtime.js` directly, so `runtime.js` remains the live shell source of truth until unified.
- Add clearer local-runtime readiness and recovery UX for the localhost provider path.
- Keep extending local-runtime readiness and recovery UX for the localhost provider path.
- Decide whether desktop should only detect localhost readiness or also manage local server lifecycle.
- Add stronger end-to-end verification for native desktop interaction beyond protocol/unit coverage.
